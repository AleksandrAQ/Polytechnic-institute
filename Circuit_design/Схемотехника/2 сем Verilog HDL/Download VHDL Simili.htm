Installation Instructions:

Download the VHDLSimili.exe file to your computer and execute (run) the file. This is a self-installing executable and will step you through the installation process. This installation includes the compiler, simulator, pre-compiled libraries for IEEE and Synopsys libraries, ready-to-compile vendor libraries, tutorials and HTML documentation.

Once you have installed VHDL Simili, online documentation is available in HTML form. After the software is installed you can use the shortcuts created by the installation program in the "Symphony EDA" program group to view the documentation, or you can find it at

file:///C:/Program Files/Symphony EDA/VHDL Simili/index.htm

Please replace "C:/Program Files/Symphony EDA" with the actual installation directory if you have chosen an installation directory that is different from the default.

You can also view most of this documentation online.

Build Updates for Release 1.4 Beta:

What will be new in Build #27 (or more): (Coming soon)

In this section, we are documenting things that will be coming soon in future builds. Changes are made to fix bugs or to add new features or to make general improvements. We cannot guarantee that all of the things listed below will be contained in the very next build. It is intended to show you what our focus areas are. Clearly, a higher priority is attributed to fixing defects.  If a particular enhancement is important to you, please contact us. The following list is maintained real time and is intended to provide our users with up to the minute information (even at the expense of airing our dirty laundry in public). Some of you who submitted bugs probably recognize the bugs you have reported.
  • In the upcoming weeks, we will be transitioning from being in Beta status to final release and calling Version 1.4 complete. This gives us the opportunity to transition V1.4 into maintenance mode and to make changes to the simulator to accommodate the needs of the upcoming GUI. Details of the design and feature-set of the GUI will be provided on this web site for your review. With respect to 1.4, we will be providing service packs once a month (if necessary). The GUI version will have a version number 2.0 since it will be considered a major release.

What is new in Build #26: April 16, 2000

  • Feature: You can now disable warnings emanating from the accelerated std_logic_arith package using the -nowarn option
  • BugFix: The simulator produced and internal error if the top level entity had unconstrained array ports.
  • BugFix: The std_logic_arith multiplication operator produced one fewer bits for a result if the types of the operands were not the same (one signed and another unsigned).

What is new in Build #25: April 3, 2000

  • Feature: You can now set generics for the entity/architecture you are simulating from the command line using the -gName=Value option. You can override the existing defaults from the command line for all scalar types and 1-dimensional character types (such as std_logic_vector, bit_vector, string, etc.)
  • Feature: The compiler now allows VHDL'87 style syntax for FILE types. You can enable VHDL'87 compatible mode using the -87 option on the compiler. When this option is enabled, the VHDL'87 syntax/semantics for FILE types are translated to their VHDL'93 equivalents. Please note that the use of this option is not recommended since the semantics of dealing with FILE types is much better defined in VHDL'93 and we recommend that you port your existing designs to the VHDL'93 style if possible.
  • Feature: VHDL Simili now supports Cypress' Warp 6.0. The scripts files have been updated to auto-detect Warp 6.0. Also, VHDL Simili no longer ships source files for the Cypress libraries. The script file has been updated to use your copy of these files.
  • Feature: VHDL Simili also ships with a GUI that aids in compilation/simulation tasks. Please visit out contributions area for more information
  • BugFix: The simulator did not correctly handle a slice to a slice assignment under the following condition -- the prefix of the two slices are the same and the array type is an array of an array and the assignment was essentially a left shift. Workaround for previous builds is to use a temporary to make the assignment.

What is new in Build #24: Mar. 19, 2000

  • BugFix: Use of a ".ALL" construct for an access type as an argument to a subprogram caused in internal error in the simulator during elaboration. Use of .ALL in other contexts was not a problem. This problem has been fixed.

What is new in Build #23: Mar. 6, 2000

  • Feature: Added an option -nostderr to the compiler and the simulator to force all output produced by these tools to go to stdout only. This is helpful in environments where it is not easy to redirect/capture both stdout and stderr to a file.
  • BugFix: The simulator produced an internal error in certain cases if the prefix of an attribute was not a simple name and if such an attribute existed in an entity/architecture with generics.
  • BugFix: The simulator required a real number to precede a time unit while reading in values of type time using textio. The simulator now allows an integer also.
